# RDR – Raindrop Distributed Resources

RDR is a powerful cross-server vault and banking solution built for serious Minecraft networks. Designed with stability, consistency, and performance in mind, RDR ensures that player inventories and town balances remain synchronized across your entire server infrastructure — without dupes, data loss, or race-condition exploits.

Built by the AMZ team after extensive testing of existing solutions, RDR focuses on one goal:

**Reliable cross-server storage without compromise.**

---

## ✨ Core Features

### 🌐 Cross-Server Player Vaults
- Seamlessly access vault contents across all connected servers.
- Move between servers without losing items.
- No duplication glitches.
- No missing items.
- No accidental vault wipes.

RDR was created after waiting over a year for cross-server functionality elsewhere — so we built it ourselves, properly.

---

### 🗄️ Database Support
Flexible persistence options to fit any infrastructure:

- H2 (Local file or in-memory)
- MySQL
- PostgreSQL

Optimized queries and safe transaction handling ensure consistency across large networks.

---

## 🔒 Stability First

The entire design philosophy of RDR revolves around:
- Atomic transactions
- Queue-based processing
- Server-safe synchronization
- Duplication prevention
- Data integrity under load

No shortcuts. No unsafe async inventory hacks. No cross-server race-condition exploits.
